# kakao-color-match-bot

카카오톡 PC 미니게임 **"정답과 같은 색을 찾아보세요!"** 를 제한 시간(라운드당 3초,
총 5라운드) 안에 자동으로 풀어주는 봇입니다. 화면을 캡처해 정답 색을 읽고, 컬러
팔레트를 드래그해 정답 색을 중앙 컬러피커로 가져옵니다.

> ⚠️ 개인 학습/자동화 실험용 프로젝트입니다. Windows 전용.

---

## 게임 메커니즘 (라이브로 확인)

- 컬러피커(마커)는 **팔레트 정중앙에 고정**되어 있고, 그 지점의 색이 곧 `선택` 색.
- 팔레트를 **드래그하면 색 판이 움직여** 중앙 마커 아래 오는 색이 바뀝니다.
  (클릭으로 마커가 이동하는 게 아니라, 판을 끌어와 원하는 색을 중앙에 맞춥니다.)
- **첫 드래그로 게임이 시작**되고, 이후 5라운드는 자동으로 진행됩니다.
- 종료 후 "게임 다시하기"로 재시작.

## 봇의 동작 원리

1. **영역 자동검출** — 매 실행 시 화면에서 게임 창(팔레트 + 정답/선택 스와치)을
   찾습니다. 팔레트 색 그라데이션을 앵커로 잡고, 중앙의 흰 마커로 진짜 팔레트를
   식별하므로 **창 위치가 바뀌어도 동작**합니다(캘리브레이션 불필요).
2. **라운드 감지** — 정답 스와치 색이 안정적으로 바뀌면 새 라운드로 인식.
3. **드래그 폐루프 제어** — 정답 색이 팔레트 어디에 있는지 찾아, 그 색을 중앙으로
   끌어오도록 드래그. 드래그 이득(gain)·부호를 매 스텝 실측(재선형화)하며 감쇠
   비례 제어로 `선택` 색을 정답에 수렴시킵니다.
4. **발산 방지** — 정답 색이 팔레트에 거의 없으면(도달 불가) gain 갱신을 막고 작은
   탐침만 해서, 엉뚱한 방향으로 폭주하지 않게 합니다.

## 설치

```
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements-dev.txt
```

의존성: `mss`(화면 캡처) · `numpy` · `opencv-python` · `keyboard`(단축키) · `pytest`(개발).

## 실행

카카오톡 색맞추기 게임 창을 띄워둔 뒤:

```
run.bat --autostart --seconds 26
```

또는 직접:

```
.venv\Scripts\python -m kcmb.main --autostart --seconds 26
```

**순서**
1. 게임 창을 화면에 띄웁니다(시작 화면이면 됨).
2. 위 명령을 실행합니다.
3. `>>> 3초 안에 카톡 게임 창을 클릭해 포커스하세요!` 가 뜨면 **게임 창을 클릭**합니다.
   (봇의 드래그가 포커스된 창으로 전달되므로 필요합니다.)
4. 봇이 게임 위치를 자동으로 찾아 시작 드래그 → 26초간 자동 플레이합니다.

> 시스템 `python`이 아니라 반드시 **`.venv\Scripts\python`(또는 `run.bat`)** 을 쓰세요.
> 라이브러리가 venv에만 설치되어 있습니다.

**F8 토글 방식**(시간 제한 없이):

```
run.bat
```

게임 창 클릭 → **F8**(시작/정지 토글) → **F9**(종료). F8/F9는 전역 단축키라
터미널이 앞에 있어도 동작합니다.

### 옵션

| 플래그 | 설명 |
| --- | --- |
| `--autostart` | 시작 시 팔레트를 드래그해 게임을 시작 |
| `--seconds N` | F8 없이 N초 동안 자동 실행(헤드리스) |
| `--focus-delay N` | 헤드리스 시작 전 게임 창을 클릭할 시간(기본 3초) |
| `--no-auto` | 자동검출을 끄고 `config.json` 고정 좌표 사용 |
| `--dry-run` | 드래그 없이 목표 좌표만 계산/로그(`debug/`에 이미지 저장) |

고정 좌표를 쓰려면 `.venv\Scripts\python -m kcmb.calibrate`로 영역을 지정해
`config.json`을 만듭니다(자동검출이 기본이라 보통 불필요).

## 테스트

```
.venv\Scripts\python -m pytest -q
```

순수 로직(색 거리·최근접 클러스터·드래그 컨트롤러·라운드 감지·영역 검출)은
화면 없이 단위 테스트로 검증합니다. 컨트롤러는 알려진 gain의 가상 색판에 대한
**폐루프 수렴 시뮬레이션**(양수/음수/큰 gain)도 포함합니다.

## 프로젝트 구조

```
kcmb/
├── config.py       # 설정 로드/저장/검증 (Region, 기본값)
├── locate.py       # 게임 영역 자동검출 (팔레트 밴드 + 마커)
├── capture.py      # mss 화면 캡처 (BGR)
├── coloralg.py     # 색 거리, 대표색, 최근접 클러스터
├── controller.py   # 부호 탐침 + 감쇠 비례 드래그 컨트롤러
├── targetstate.py  # 정답색 디바운스 → 라운드 감지
├── input_win.py    # Win32 SendInput 드래그(모션 프로파일) + settle
├── hotkeys.py      # F8 arm / F9 quit
├── dpi.py          # DPI-aware 설정
├── calibrate.py    # (선택) 수동 영역 캘리브레이션
├── probe.py        # (선택) 드래그 반응 실측
└── main.py         # 진입점: 폐루프 배선
```

## 상태 / 실측 결과

동작 확인 완료. 자동 플레이로 **5라운드 861~967점**, 라운드별 대부분 96~99%
정확도. 어둡거나 저채도인(팔레트에 없는) 정답 색은 도달 한계로 낮게 나올 수
있습니다.

## 주의

- 카카오톡과 봇은 같은 권한(둘 다 비관리자 권장)으로 실행해야 입력이 전달됩니다.
- `config.json`(기기별 좌표)과 `debug/`(디버그 이미지)는 커밋 대상이 아닙니다(.gitignore).
